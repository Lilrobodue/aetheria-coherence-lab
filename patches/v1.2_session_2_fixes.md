## Change details

### 1. Harmonic lock (`cross-regime-v1.2.js`)

Three precise changes:

- **1:1 ratio removed from `HARMONIC_RATIOS`.** "Same frequency" isn't a harmonic relationship — it's sameness. The harmonic list now contains only non-trivial integer ratios whose numerator and denominator both have digital roots in {3, 6, 9}: 2:1, 3:1, 3:2, 4:3, 5:3, 6:1, 6:5, 9:1, 9:2, 9:4, 9:5, 9:7, 9:8.

- **Minimum frequency separation enforced.** A new `MIN_FREQ_SEPARATION_HZ = 0.01` constant. Pairs closer than this are excluded from the harmonic analysis entirely — not counted as harmonic AND not counted as non-harmonic. They're sameness, which is neither.

- **Tolerance tightened from ±2% to ±1.5%.** Session 2 data showed false positives near ratio boundaries; the tighter window rejects accidental alignment while still catching real harmony.

The combined effect: the harmonic lock will now show a much wider range of values — probably mostly 0 with occasional spikes to 0.33, 0.5, or 1.0 when the regime envelopes actually enter harmonic relationships. When you see it climb during a session, that climb will mean something.

### 2. Arousal anchor (`arousal-anchor-v1.2.js`)

The v1.1 version only evaluated drift at PIVOT points. The v1.2 version evaluates continuously at 1 Hz during ENTRAIN, with three compensating safeguards to prevent thrashing:

- **`updateDrift(features, timestamp)`** is called every second by the coherence engine with the current feature snapshot. It computes the live arousal index, appends to a 30-second rolling drift history, and checks whether drift has sustained beyond the threshold.

- **`flip_margin` tightened from 0.15 to 0.10.** The session 2 drift was clearly meaningful but may not have sustained above 0.15. A 0.10 threshold responds to real shifts faster.

- **Sustained-drift requirement: 15 consecutive seconds.** A flip only queues if the drift has been beyond the threshold in the same direction for 15 seconds running. Single-second excursions don't trigger. Brief noise can't flip the cascade. Only a real state change that persists can.

- **`consumePendingFlip()`** is called by the state machine at decision points. If a flip has been queued, it's applied and the drift history is cleared (so the next flip requires a fresh sustained drift, preventing immediate re-flipping on the same data).

- **Graceful null handling:** arousal index computation handles missing head features correctly. If the Muse is offline, the arousal index is computed from heart terms alone and scaled appropriately, rather than returning null or a default.

### 3. Selection rules (`selection-rules-v1.2.js`)

This is the biggest architectural change in the bundle — the introduction of a `CascadeCursor` class that tracks where in the root cycle the user currently is as session state.

Previously, each PIVOT recomputed the starting root from scratch based on direction defaults. In v1.2:

- **CascadeCursor** is a new session-scoped object that holds the current direction and position within the cascade order.
- **Cascade order** is explicit: ASCENDING = [3, 6, 9], DESCENDING = [9, 6, 3]. Each selection advances the cursor by one step.
- **PIVOT selections** advance the cursor the same way ADVANCE does. The only difference between PIVOT and ADVANCE is which regime the new frequency comes from — the root progression is continuous.
- **Direction flips** reset the cursor to position 0 for the new direction (`flipTo(newDirection)`). When the cascade flips from DESCENDING to ASCENDING, the user experiences the new phase cleanly starting at root 3.
- **Octave climbing within HEAD** is preserved — ADVANCE within HEAD still walks up the octave ladder.
- **No-repetition window of 5** is preserved in the library lookup with a three-tier fallback (exact match → same regime+root → same regime).

The result: a descending cascade will walk 9 → 6 → 3 → 9 → 6 → 3 ... with each selection, regardless of whether the individual selections are ADVANCE or PIVOT. An ascending cascade walks 3 → 6 → 9 → 3 → 6 → 9. The breath metaphor is honored.

## Wiring it together

The key integration change is in the coherence engine and state machine. The coherence engine needs to call `anchor.updateDrift()` every second during ENTRAIN. The state machine needs to create a `CascadeCursor` at session start, pass it to the selection functions, and call `anchor.consumePendingFlip()` at each decision point.

```javascript
// In coherence-vector.js (CoherenceEngine)
import { ArousalAnchor } from '../policy/arousal-anchor.js';

class CoherenceEngine {
  constructor(bus, config) {
    this.anchor = new ArousalAnchor(config.policy);
    // ... other setup
  }

  _tick() {
    // ... existing feature computation ...

    // v1.2: continuous drift monitoring during ENTRAIN
    if (this.currentState === 'ENTRAIN' && this.anchor.anchor != null) {
      const t = performance.now() / 1000;
      this.anchor.updateDrift(latestFeatures, t);
    }
  }

  setBaselineAnchor(baselineFeatures) {
    return this.anchor.setAnchor(baselineFeatures);
  }
}
```

```javascript
// In state-machine.js (PolicyEngine)
import { CascadeCursor } from '../policy/selection-rules.js';
import { selectFirstFrequency, selectNextFrequency, onDirectionFlip }
  from '../policy/selection-rules.js';

class PolicyEngine {
  constructor(bus, config, library) {
    this.cursor = new CascadeCursor();
    // ... other setup
  }

  _onBaselineComplete() {
    const { initialDirection } = this.coherence.setBaselineAnchor(baselineFeatures);
    this.cursor.reset(initialDirection);
    // transition to ASSESS ...
  }

  _assess() {
    // Check if the anchor has queued a direction flip
    const newDirection = this.coherence.anchor.consumePendingFlip();
    if (newDirection) {
      onDirectionFlip(newDirection, this.cursor);
      this._log(`Cascade flipped to ${newDirection} based on sustained drift`);
    }

    // Now select the next frequency
    const next = selectNextFrequency(
      this.lastHint,
      this.currentFreq,
      this.latestCoherence,
      this.coherence.anchor.anchor,
      this.library,
      this.history,
      this.cursor
    );

    // transition to PRESCRIBE ...
  }
}
```
